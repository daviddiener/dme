info What does this application do?

DME is a prototypical tool for modeling business processes and data. The goal is to automatically extract data models based on business process models. The designer tool can be used to create the business process models, which are based on an advanced type of Petri nets (as shown on the next page). Using this Petri net, different extraction rules are applied to extract the data model. The resulting data model serves as a foundation for software development by providing a representation of the system's data structure. This, in turn, enables developers to make informed decisions during the early phases of the software development life cycle, speeding up the overall development process.

The Designer tool allows for the modeling of such business process models. They process models are based on an advanced type of Petri Nets.

On the basis of this Petri Net, the application will extract a data model , following a set of rules for the extraction of the respective concepts.

 What are Petri Nets?

A Petri net (Carl Adam Petri in 1962, Kommunikation mit Automaten) consists of two main components: places and transitions. Places are represented by circles and can contain tokens, which are small dots that indicate the presence of a resource or a state. Transitions are represented by rectangles and define the actions or events that can occur in the system. Arcs connect places and transitions and indicate the flow of tokens between them.

The behavior of a Petri net is determined by its initial marking, which is the distribution of tokens among the places at the start of the system, and the firing rule, which defines when and how transitions can fire. When a transition fires, it consumes tokens from its input places and produces tokens in its output places, which may trigger other transitions to fire.

Petri nets can be used to model a wide range of systems, including business processes, software systems, and networks. They provide a graphical and intuitive way to represent complex systems, and they can be analyzed using various techniques to verify their correctness, performance, and other properties.

The image below shows a Petri net model for a barber shop. The net includes a place waiting that models a waiting area where customers wait for the barber to serve them, and two places that model the barber’s status, busy or idle. The transitions model events that can occur: A firing of transition enter corresponds to a customer entering the waiting area of the shop. Transition serve corresponds to a customer moving to the barber’s chair for service. Transition done corresponds to the customer leaving the barber chair after receiving service, with the barber becoming idle.This modelof the barber shop permits any number of customers to enter the waiting room, but models the constraint that only one customer can occupy the barber chair. This net is an unbounded Petri net because the number of tokens in the waiting place can increase indefinitely.

Design or Import a Petri Net

The first step is to design or import a Petri net diagram using the Designer tool provided in this application or by importing an example Petri net from a list of resources available on the internet. To design a Petri net, go to the Designer and use the tools provided to add places, transitions, and arcs. To import a Petri net, select an example from this page or press the "Import" button and paste the Petri net diagram from the clipboard.

Define Token Schemas and Roles

In this step, define token schemas for each place in the Petri net. Token schemas include the name, superclasses, and attributes of the token. To define a token schema, select the place and enter a token schema name. The field for the superclass and attribute definitions will appear, allowing you to enter the token schema information. If you do not define a token schema, the place name will be used for the data model generation in Step 4.

You can also define roles for each transition. A role represents a person, organization, or entity responsible for the occurrence of the transition. To define a role, select the transition and enter the name of the role in the role field. The roles will be converted into classes in the data model, and each class will be assigned the attributes "id", given that the class is not further specified by a token schema with the same name as the role.

 Execute Data Model Extraction

After completing the previous steps, you can extract a data model by going to the data model extraction page. The application will automatically generate a UML class model for you based on the Petri net model and token schemas and roles defined in Step 4.

The resulting data model can be used for the early phases of software development. Congratulations! You have successfully modeled a Petri net diagram and extracted a data model from it.
