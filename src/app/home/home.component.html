info Design or Import a Petri Net

The first step is to design or import a Petri net diagram using the Designer tool provided in this application or by importing an example Petri net from a list of resources available on the internet. To design a Petri net, go to the Designer and use the tools provided to add places, transitions, and arcs. To import a Petri net, select the "Import" button and paste the Petri net diagram from the clipboard.

 Define Token Schemas for Places

In this step, define token schemas for each place in the Petri net. Token schemas include the name, superclasses, and attributes of the token. To define a token schema, select the place and enter a token schema name. The field for the superclass and attribute definitions will appear, allowing you to enter the token schema information. If you do not define a token schema, the place name will be used for the data model generation in Step 4.

 Define Roles for Transitions

In this step, define roles for each transition. A role represents a person, organization, or entity responsible for the occurrence of the transition. To define a role, select the transition and enter the name of the role in the role field. The roles will be converted into classes in the data model, and each class will be assigned the attributes "id" and "name."

 Execute Data Model Extraction

After completing Steps 1-3, you can extract a data model by going to the data model extraction page. The application will automatically generate a UML class model for you based on the Petri net model and token schemas and roles defined in Steps 2 and 3. The resulting data model can be used for the early phases of software development.

Congratulations! You have successfully modeled a Petri net diagram and extracted a data model from it.
